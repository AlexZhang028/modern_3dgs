# 全局体素统计共现率：用于提取静态背景的未来可行方案 (Global Voxel Statistical Co-occurrence)

## 1. 背景与动机
在多帧点云（如 SelfCap 数据集）初始化以及 FreeTimeGS 的动态建模过程中，精确剥离静态背景和动态前景至关重要。
当前初始化脚本（`process_selfcap_pcd.py`）采用了基于局部 KNN 匹配求速度的启发式策略。为了弥补单纯基于距离的缺陷，我们引入了基于颜色的相关匹配（Color-aware KNN，方案A）和空间连贯性中值滤波（Spatial Median Smoothing，方案C）。
虽然方案A和C大大改善了散粒噪声，但这依然是局部且仅依赖相邻两帧的信息进行判断。真实点云存在孔洞、遮挡及传感器噪声，偶尔的大幅度闪烁可能会穿透这种局部过滤器。

**方案D（全局体素统计共现率）** 是一种基于全局时序统计特征的背景分离方法。它可以彻底解决因为单帧缺失导致的偶然大位移错配问题，属于相对长期的底层优化方案。

## 2. 核心原理
**核心思想**：物理静态背景在整个时间序列中会在相同的空间位置持续“共现”。动态物体则是短暂停留在某些位置。

1. **构建全局体素网格（Voxel Grid）**
   将初始化阶段使用的所有帧中的点云囊括在一起，划分出一个低分辨率（或可配置分辨率，比如 3cm × 3cm × 3cm）的全场景体素网格。
2. **多帧映射与计数 (Frame Co-occurrence Counting)**
   对于体素网格的每一个格子 $V_{x,y,z}$，遍历时间序列的每一帧 $f$。如果帧 $f$ 中有任意一个点落入这个格子，则认为在这帧中该格子被“占据”。
   统计每个格子被占据的绝对帧数 $N_{occ}$。
3. **高频共现筛选**
   设定一个阈值（例如：$N_{occ} > 0.8 \times \text{总帧数}$）。满足这个阈值的格子，从时序上明确证明了它是一个持久存在的结构，即**绝对静态背景**。
4. **反向过滤与标记**
   最后再遍历每一帧的原始观测点云：
   - 只要这个点落入了被标记为**静态背景**的体素中，它就被强制赋予 $Velocity = (0, 0, 0)$ 平息其伪动态，或者只在第0帧保留它而在后续帧中丢弃它以防冗余累计。
   - 反之，落入低频共现区（被判定为动态区）的点才进行真实的 KNN 速度估计和运动点建模。

## 3. 为什么将其作为未来优化（优缺点）
### 优点
* **极致的鲁棒性**：完全不依赖于相邻帧脆弱的对应点匹配，避免了局部噪声和剧烈遮挡带来的误判。
* **物理意义明确**：利用整段视频的长程时间线索判定动静，理论上限更高。

### 缺点 / 现有阻力
* **显存与内存要求较高**：对于非常大的场景（如室外大尺度），对所有帧做全局的网格化和计数需要更精细的数据结构（如 Hash Voxel 或 Sparse Tensor）。
* **可能切掉缓慢移动的物体**：如果主体运动非常缓慢（占据同一个格子的时间超过阈值），也会被误判为背景。

## 4. 实施路径 (Implementation Roadmap)

如果要在未来的 `process_selfcap_pcd.py` 或特定的预处理 pipeline 中引入此技术，建议流程如下：
1. **预计算包围盒**：一次 Pass 扫完所有需要帧，获取全局 AABB (Axis-Aligned Bounding Box)。
2. **定义分辨率**：根据 AABB 和指定 `voxel_size` 初始化一个 Hash Map（在 Python 中可以用 `dict` 或是 `open3d.geometry.VoxelGrid`）。
3. **第一次统计**：第二次 Pass 扫帧，把 `(point_x // voxel_size, ...)` 作为 Key，存入每个体素占据了哪些帧（集合 `set()` 以去重）。
4. **生成背景掩码**：提取出属于背景的 Voxel Key Set。
5. **第二次生成**：真正的生成步骤。遍历时一旦点坐标落入 Voxel Key Set 中，直接跳过 KNN 速度计算，以背景的身份处理。
